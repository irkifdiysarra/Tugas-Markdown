Workflow

repositori lokal terdiri dari 3 bagian pokok yang disebut trees.

Nomor Nama Fungsi
1. Working Directory Untuk menyimpan berkas asli
2. Index Untuk pengelolaan data
3. Head Untuk mengarahkan ke commit terakhir

Add and Commit

Nomor Sintaks Fungsi
1. $ git add <namaberkas> $ atau git add * Melakukan perubahan dengan menambahkan direktori di indeks.
2. $ git commit -m 'pesan terhadap perubahan yg dilakukan' untuk commit seluruh perubahan pada berkas

Penjelasan :

(-m) pada sintaks diatas maksudnya message / pesan, isi pesan ada di dalam tanda kutip satu (‘’) atau kutip (“”). Dengan melakukan commit maka kita harus meninggalkan pesan agar perubahan yg terjadi di repository dapat dipantau. Setelah commit berkas sudah ada di bagian Head.

Pushing Changes

Nomor Sintaks Fungsi
1. $ git push origin <branch> Untuk mengirimkan berkas ke remote repositori
2. $ git remote add origin <server> Sekarang kamu bisa mengirimkan perubahan ke server jarak-jauh yang dituju

Penjelasan :

Saat ini perubahan telah tersimpan di HEAD salinan kerja lokal. Ubah sesuai dengan branch yang diinginkan.
Jika repositori yang ada belum dikloning dan ingin dihubungkan ke remote server, perlu ditambahkan sintaks no.2

Branching

percabangan atau branching digunakan untuk mengembangkan fitur-fitur secara terisolasi. Cabang utama atau master merupakan cabang bawaan ketika kamu membuat repositori.
Gunakan cabang lain untuk melakukan pengembangan, setelah selesai, gabungkan / merge kembali ke cabang utama.

Nomor Sintaks Fungsi
1. $ git checkout -b fiturX buat cabang baru dengan nama “fiturX”
2. $ git checkout master beralih kembali ke master
3. $ git branch -d fitur_x hapus cabang yang tadi dibuat
4. $ git push origin <cabang> suatu cabang tidak terbuka untuk yang lainnya kecuali jika kamu mengirimkannya ke remote repositori.

Penjelasan :

Membuat cabang / branch baru, kemudian masuk ke dalam branch fiturX dengan sintaks no.1 dan menjadikan fiturX sebagai working page.
Beralih ke branch master dengan menggunakan sintaks no.2 (menjadikan master sebagai working page).
Sintaks no.3 digunakan untuk menghapus branch yg sudah dibuat.

Update and Merge

Nomor Sintaks Fungsi
1. $ git pull untuk memperbarui repositori lokal ke komit terkini
2. $ git merge <cabang> mengambil dan menggabungkan perubahan direktori kerja dengan cabang / branch lain (misal master)

Penjelasan :

pada kasus diatas, git mencoba menggabungkan perubahan secara otomatis.
Sayangnya hal ini tak selalu berjalan mulus dan bisa menyebabkan konflik.
Jika terjadi konflik maka menyelasaikan konflik tersebut secara manual dengan menyunting berkas yang ditunjukkan git.
Setelah itu, perlu dimarkahi dengan sintaks berikut :

$ git add <namaberkas>

Nomor Sintaks Fungsi
3. $ git diff <cabang_asal> <cabang_tujuan> melakukan pratinjau terhadap cabang / branch sebelum menggunakan merge

Tagging

Nomor Sintaks Fungsi
1. $ git tag 1.0.0 1b2e1d63ff membuat penanda baru dengan nama 1.0.0

Penjelasan :

Dianjurkan membuat penanda atau tags untuk perangkat lunak yang dirilis.
1b2e1d63ff adalah 10 karakter pertama dari identitas komit sebagai referensi ke penanda.
Kamu bisa mendapatkan identitas komit dengan melihat…

Log

Nomor Sintaks Fungsi
1. $ git log Melihat aktifitas yg terjadi dalam repositori git
2. $ git log --author=bob Menambah parameter untuk melihat commit dari penulis tertentu, misal : bob
3. $ git log --pretty=oneline Untuk melihat log yang dimampatkan secara satu baris per commit
4. $ git log --graph --oneline --decorate --all melihat ASCII tree art seluruh branch disertai nama dan penandanya
5. $ git log --name-status Melihat perubahan berkas
6. $ git log --help Untuk melihat sintaks-sintaks yg terdapat di git.

Penjelasan :

kamu bisa menambahkan banyak parameter untuk menampilkan log sesuai keinginan. Contohnya ada di sintaks no.1

Replace Local Changes

Nomor Sintaks Fungsi
1. $ git checkout -- <namaberkas> Seandainya kesalahan terjadi bisa dikembalikannya menggunakan sintaks ini

Penjelasan :

perintah di atas mengembalikan perubahan di dalam berkas kerja dengan konten terakhir dari HEAD. Perubahan dan berkas baru yang telah ditambahkan ke indeks akan tetap tersimpan.
Jika kamu ingin menggugurkan perubahan dan commit lokal secara menyeluruh, ambil riwayat terakhir dari server dan arahkan ke cabang master lokal seperti sintaks berikut ini :

$ git fetch origin / $ git reset --hard origin/master

Useful Hints

Nomor Sintaks Fungsi
1. $ gitk GUI git bawaan
2. $ git config color.ui true menggunakan output git penuh warna
3. $ git config format.pretty oneline menunjukkan log satu baris per komit
4. $ git add -i menggunakan penambahan interaktif
